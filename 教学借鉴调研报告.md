# 财务软件教学模式调研报告

**调研对象**：金蝶KIS教学版、用友U8教学版、QuickBooks Online、Xero + 教育游戏化研究
**目的**：借鉴业界做法，优化本系统的教学体验

---

## 一、业界做法对比

### 金蝶KIS教学版 / 用友U8教学版

| 做法 | 说明 | 对我们的启示 |
|------|------|-------------|
| **实验账套体系** | 每个实验提供"准备账套"（前置数据）和"结果账套"（答案），可独立也可衔接 | ⭐ 我们也需要"续接账套"——从4月开始学的学生能预置1~3月数据 |
| **分步引导** | 业务描述 → 岗位说明 → 操作指导，每一步配截图 | ⭐ 当前引导模式一次性全填完，且分步引导是预留清单第1项 |
| **原始单据驱动** | 以发票、银行回单等真实凭证为任务入口 | ✅ 我们已有VoucherDisplay，与业界一致 |
| **模块化拼板** | 26/56/80课时可组合，教师灵活选用 | 长期可参考，当前先按月线性推进 |
| **教学版限制** | 功能完整但限制3个月账期 | 不适用（我们是纯教学） |
| **云实训平台** | 演示/教程/练习/测试四种场景 | ✅ 我们三种模式类似（引导/练习/考试） |

### QuickBooks Online / Xero

| 做法 | 说明 | 对我们的启示 |
|------|------|-------------|
| **Demo company** | 预置一家完整演示公司，新用户直接上手 | ✅ initTeachingAccount已类似 |
| **分层学习路径** | Beginner → Intermediate → Advanced | ⭐ 可以给任务标难度，学生按进度解锁 |
| **银行数据自动匹配** | Bank feeds自动勾对，系统推荐匹配 | 技术亮点，但属于系统功能而非教学 |
| **首次引导（Guided Tour）** | 第一次使用时逐高亮引导操作 | ⭐ 可以考虑首次进入教学时做操作指引 |

### 教育游戏化研究（Gamification）

| 做法 | 说明 | 对我们的启示 |
|------|------|-------------|
| **成就/徽章系统** | 里程碑奖励，如"连续答对10题" | ⭐ 预留清单有，可以设计 |

| **渐进难度** | 从简单到复杂递增 | ✅ 1月简单→2月进阶→3月季末→4月新业务，需要标注难度 |

## 三、最推荐当前做的（具体方案）

### 方案A：任务难度标注 + 筛选（~50行，数据改动为主）

**做什么**：每个教学任务加 `difficulty: 1|2|3` 字段（1简单/2中等/3困难），教学中心可筛选。

**改动文件**：
- `year1.js` + `02.js` + `03.js` + `04.js` — 每个任务加 `difficulty`
- `TutorialCenter.vue` — 筛选按钮组
- `tutorial.test.js` — 验证字段存在

**效果**：学生看到"★ 简单 6个 / ★★ 中等 15个 / ★★★ 困难 9个"，从简单开始建立信心。

### 方案B：分步引导（重构引导模式，~200行）

**做什么**：当前引导模式一次性预填所有分录，改为：

```
第1步：[看凭证] 展示电子凭证（发票/回单）→ 点击"下一步"
第2步：[填科目] 只填第一行借方科目 → 验证 → 正确才能继续
第3步：[填金额] 只填借方金额 → 验证
第4步：[填第二行] 贷方科目 → 验证
第5步：[填贷方金额] → 验证
第6步：[总体验证] 借贷平衡检查 + 提交
```

**改动文件**：
- `VoucherEntry.vue` — 步进状态机 + 逐题验证（核心逻辑）
- `VoucherEntryTable.vue` — 支持"只解锁当前行"模式

**教学效果**：学生不再"一看就会、一做就废"，每步都能获得即时反馈。

---

### 我的建议

如果你优先追求 **学习效果质变** → 选方案B（分步引导），这是金蝶/用友教材的核心特色，也是我们预留清单最高优先级。

如果你优先追求 **快速提升体验** → 选方案A（难度标注），50行代码就能做出明显改善。

两者不冲突——方案A可以马上做，方案B等5月数据做完后投入更多时间重构。
